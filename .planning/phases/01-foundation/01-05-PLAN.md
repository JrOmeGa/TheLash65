  <action>
Two changes to DialogContent in src/components/ui/dialog.tsx:

1. REMOVE centering transforms from the base className string on line 39. Delete these exact classes:
   - `left-[50%]`
   - `top-[50%]`
   - `translate-x-[-50%]`
   - `translate-y-[-50%]`
   Also remove the slide animations that reference those positions:
   - `data-[state=closed]:slide-out-to-left-1/2`
   - `data-[state=closed]:slide-out-to-top-[48%]`
   - `data-[state=open]:slide-in-from-left-1/2`
   - `data-[state=open]:slide-in-from-top-[48%]`

   The REMAINING base classes should be:
   `"fixed z-50 grid w-full max-w-lg gap-4 bg-[var(--color-surface-container-lowest)] p-6 shadow-floating duration-200 data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 data-[state=closed]:zoom-out-95 data-[state=open]:zoom-in-95 sm:rounded-lg"`

   WHY: tailwind-merge cannot cancel `left-[50%]` and `top-[50%]` when a consumer passes `inset-0`, because they are different utilities. Removing them lets GalleryLightbox's `inset-0` work correctly.

2. REMOVE the entire auto-injected DialogPrimitive.Close block (lines 45-51) that renders after {children}. The DialogContent should render ONLY {children} inside the DialogPrimitive.Content element.

   WHY: GalleryLightbox provides its own DialogClose with white text for dark backgrounds. The auto-injected one renders on top with default styling, making the custom close button invisible.

After the edit, DialogContent should look like:

```tsx
const DialogContent = React.forwardRef<
  React.ElementRef<typeof DialogPrimitive.Content>,
  React.ComponentPropsWithoutRef<typeof DialogPrimitive.Content>
>(({ className, children, ...props }, ref) => (
  <DialogPortal>
    <DialogOverlay />
    <DialogPrimitive.Content
      ref={ref}
      className={cn(
        "fixed z-50 grid w-full max-w-lg gap-4 bg-[var(--color-surface-container-lowest)] p-6 shadow-floating duration-200 data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 data-[state=closed]:zoom-out-95 data-[state=open]:zoom-in-95 sm:rounded-lg",
        className
      )}
      {...props}
    >
      {children}
    </DialogPrimitive.Content>
  </DialogPortal>
));
```

GalleryLightbox.tsx requires NO changes -- it already passes the correct fullscreen classes and its own DialogClose.

No other components in the codebase consume DialogContent, so there are no other callers to update.
