## Spacing Scale

Declared values (multiples of 4, 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Between step indicator dots, between slot buttons in grid |
| md | 16px | Default section padding, card inner padding, form field gaps |
| lg | 24px | Between wizard sections, between service cards |
| xl | 32px | Page-level top padding inside scroll container |
| 2xl | 48px | Wizard container vertical inset from header |
| 3xl | 64px | Reserved — not used in this phase |

Exceptions:
- Touch targets (date pills, slot buttons, toggle rows) minimum height **44px** — iOS/Android accessibility minimum
- Date strip horizontal padding: **16px** left/right to prevent pills clipping at viewport edge
- Slot grid gap: **8px** horizontal, **8px** vertical (uniform 2–3 column grid)

## Typography

All sizes carry forward patterns established in ServiceCard and Header (Phase 1 codebase).

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | `--font-heading` (Noto Serif) | Confirmation page booking summary heading |
| Heading | 20px | 600 (semibold) | 1.2 | `--font-heading` (Noto Serif) | Step heading ("Choose a Date", service card names) |
| Body | 16px | 400 (regular) | 1.5 | `--font-body` (Manrope) | Wizard step descriptions, next-steps copy on confirmation |
| Label | 14px | 400 (regular) / 600 (semibold) | 1.5 standard / 1.3 badge | `--font-body` (Manrope) | Service descriptions, price display, slot time labels (400); eyebrow labels, "UNAVAILABLE" badge, day names in date strip, duration labels (600 semibold at line-height 1.3) |

Only 2 weights are used: **400 (regular)** and **600 (semibold)**. No 500 or 700.

Note: The former Micro role (12px) is consolidated into Label (14px semibold). Badge/eyebrow/day-name contexts use 14px at weight 600 and line-height 1.3 to preserve visual hierarchy without a fifth type size.

## Color

All color values sourced from `src/app/globals.css` `@theme` block.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fff8f5` (`--color-surface`) | Page background, wizard container background |
| Secondary (30%) | `#f5ede6` (`--color-surface-container-low`) | Date pills (unselected), slot buttons (unselected), admin table rows, confirmation card background |
| White surface | `#ffffff` (`--color-surface-container-lowest`) | Service selection cards, confirmation booking summary card |
| Accent (10%) | `#755944` (`--color-accent`) | gradient `#755944 → #9c7660` | Selected date pill fill, selected slot button fill, primary CTA button background, confirmation heading, price display, active step indicator |
| On-surface text | `#1F1B18` (`--color-on-surface`) | All body text, heading text |
| Muted text | `rgba(31, 27, 24, 0.60)` | Unavailable slot labels, date strip day names, admin toggle off-state labels |
| Destructive | `#b91c1c` (`--color-destructive`) | "Unblock date" button, destructive admin actions only |

Accent reserved for:
1. Selected date pill in the date strip (filled background)
2. Selected time slot button (filled background)
3. Primary CTA buttons: "Confirm Booking", "Book this date" (desktop), "Block this day"
4. Active wizard step indicator dot
5. Price text in service selector step
6. Confirmation page heading text

Accent is NOT used for: disabled states, informational text, borders, background fills other than the 6 elements above.

Unavailable slots (grayed state): `--color-surface-container-low` background, `rgba(31, 27, 24, 0.35)` text, `pointer-events: none`, no hover state.

## Component Specifications

### Date Strip (DateStrip.tsx)

DateStrip is the primary visual anchor on Step 1 — it spans full width below the step heading and receives the first user interaction.

- Horizontal scroll container, `overflow-x: auto`, `-webkit-overflow-scrolling: touch`, `scroll-snap-type: x mandatory`
- Each pill: `scroll-snap-align: start`, min-width `60px`, height `44px`, `border-radius: 8px`
- Pill layout: day name (**14px label, semibold 600, line-height 1.3**, muted) stacked above date number (16px body, semibold when selected)
- **Available, unselected:** background `--color-surface-container-low`, text `--color-on-surface`
- **Available, selected:** background `linear-gradient(135deg, #755944 0%, #9c7660 100%)`, text `#ffffff`
- **Unavailable (D-08):** background `--color-surface-container-low`, text `rgba(31, 27, 24, 0.35)`, `pointer-events: none`
- Gap between pills: `8px`
- Show today as first pill; strip spans 60 days (D-03)
- Strip has no visible scrollbar on mobile (`scrollbar-width: none`)

### Time Slot Grid (TimeSlotGrid.tsx)

- CSS grid: `grid-template-columns: repeat(3, 1fr)` on mobile; fallback to 2 columns if fewer than 3 slots exist
- Gap: `8px` both axes
- Each slot button: height `44px`, `border-radius: 4px`, `font-size: 14px`, `font-weight: 600`
- **Available, unselected:** background `--color-surface-container-low`, text `--color-on-surface`, border `none`
- **Available, selected:** background `linear-gradient(135deg, #755944 0%, #9c7660 100%)`, text `#ffffff`
- **Unavailable (D-07):** background `--color-surface-container-low`, text `rgba(31, 27, 24, 0.35)`, `pointer-events: none`; label below time reads "Unavailable" at **14px label, semibold 600** — do NOT hide these buttons
- Focus ring: `outline: 2px solid #755944; outline-offset: 2px` (matches existing button.tsx pattern)

### Service Selector (ServiceSelector.tsx)

- Reuse existing `ServiceCard` component exactly as established in Phase 1
- Selected card state: add `outline: 2px solid #755944` around card boundary, no change to interior
- Card interaction: `active:scale-[0.98]` (already in ServiceCard className)

### Booking Wizard Step Indicator

- Linear dots or numbered pills at top of wizard
- 4 steps: Date / Time / Service / Confirm
- Active step: filled circle `#755944`, 10px diameter
- Completed step: filled circle `#9c7660`, 10px diameter
- Upcoming step: outlined circle `rgba(117, 89, 68, 0.30)`, 10px diameter
- Connecting line between dots: `1px solid rgba(117, 89, 68, 0.20)`
- Step label text: **14px label, semibold 600, line-height 1.3**, muted — shown on desktop only (hidden on mobile to save space)

### Confirmation Page (book/confirmation/[id])

- Auth-gated, full-page layout under standard header
- Booking summary card: white `#ffffff` background, `border-radius: 12px`, `shadow-elevated`, `padding: 24px`
- Card heading: 28px Display, semibold, `--font-heading`, color `#755944`
- Summary rows (service, date, time, price): 16px body label on left, 14px label value on right, `border-bottom: 1px solid rgba(31,27,24,0.06)` between rows
- Next steps copy block: 16px body, `--color-on-surface` at 75% opacity, `margin-top: 24px`
- QR placeholder: `--color-surface-container-low` background, `border-radius: 8px`, `padding: 32px`, centered — contains 14px label copy (see Copywriting Contract)

### Admin Schedule Page (/admin/schedule)

- Full-width page under admin layout
- Weekly hours table: 7 rows, no visible outer border — rows separated by `border-bottom: 1px solid rgba(31,27,24,0.06)`
- Each row: day name (16px body, semibold, 80px column), On/Off toggle (shadcn-style toggle, accent color when on), open-time input (14px label), close-time input (14px label)
- Toggle off state: `--color-surface-container-low` track, muted text
- Toggle on state: `#755944` track fill
- Block date section below weekly table — separated by `margin-top: 32px`
- Block date UI: date display area (pill style, same as date strip pill) + "Block this day" button (variant `destructive`, `h-11`)
- Blocked dates list: below block form, each blocked date as a removable chip — chip background `rgba(185, 28, 28, 0.08)`, text `#b91c1c`, remove `×` icon at 14px

### Loading / Skeleton States

- Date strip loading: 7 skeleton pills (`--color-surface-container-low`, `border-radius: 8px`, 60×44px each, animated `opacity` pulse at 1s ease-in-out)
- Slot grid loading: 6 skeleton rectangles in 3-column grid, same animation
- Service cards loading: 2 skeleton cards matching ServiceCard dimensions
- Animation: `opacity: 0.5 → 1.0` pulse, 1s ease-in-out, `animation-iteration-count: infinite` — no background shimmer to keep implementation simple

## Accessibility Contract

- All interactive elements: minimum `44×44px` touch target (iOS/Android guideline)
- Focus ring: `outline: 2px solid #755944; outline-offset: 2px` — all interactive elements (matches existing button.tsx)
- Unavailable slots: `aria-disabled="true"` + `aria-label="[time] Unavailable"` — screen reader announces state
- Date strip: `role="listbox"` container, each pill `role="option"` with `aria-selected`
- Slot grid: `role="group"` with `aria-label="Available times"`, each button `aria-pressed` when selected
- Wizard step indicator: `aria-label="Step [N] of 4: [step name]"` on active step
- Error messages: `role="alert"` so screen readers announce on insertion
- Confirmation page: `<h1>` for "Booking Confirmed" — no heading hierarchy issues
